# Contributing to the OOS Spec

## How to contribute

1. **Publish your OOS** — The best contribution is a real OOS from a real team. Visit [orgtp.com/publish](https://orgtp.com/publish).

2. **Improve the spec** — Open an issue if something in the spec doesn't match your experience running agents in production.

3. **Build an integration** — Create an OTP plugin for your agent framework and submit a PR to add it to the integrations list.

## Guidelines

- Be specific. "Agents should coordinate" is not a contribution. "When Agent A writes to shared state, Agent B should read it within 60 seconds or flag stale data" is.
- Share scars, not theories. The most valuable contributions come from failures in production, not ideas about what might work.
- Protect secrets. Never include API keys, credentials, client names, or proprietary business logic in an OOS contribution.

## Code of conduct

Cause no harm. Leave the world better than you found it.
